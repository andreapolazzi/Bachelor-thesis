Define feature sets

feature_sets <- list(
  full = model_data,
  no_tel_insert = model_data[, !names(model_data) %in% "telomere_insertion_rate"],
  no_tf = model_data[, !names(model_data) %in% "tf_rate"],

  no_tel_content = model_data[, !names(model_data) %in% "telomere_content_log2"],

Train models

library(caret)
library(pROC)
library(dplyr)
library(randomForest)

results <- data.frame()
all_resamples <- list()

for (fs in names(feature_sets)) {
  
  df <- feature_sets[[fs]]
  
  log_model <- train(
    alt_status ~ .,
    data = df,
    method = "glm",
    family = "binomial",
    metric = "ROC",
    trControl = ctrl
  )
  
  rf_model <- train(
    alt_status ~ .,
    data = df,
    method = "rf",
    metric = "ROC",
    trControl = ctrl
  )
  
  models <- list(LogReg = log_model, RF = rf_model)
  
  for (m in names(models)) {
    
    model <- models[[m]]
    
    rs <- model$resample
    
    rs$feature_set <- fs
    rs$model <- m
    
    all_resamples[[paste(fs, m, sep = "_")]] <- rs
    
    results <- rbind(
      results,
      data.frame(
        feature_set = fs,
        model = m,
        mean_AUC = mean(rs$ROC, na.rm = TRUE),
        sd_AUC = sd(rs$ROC, na.rm = TRUE),
        mean_sensitivity = mean(rs$Sens, na.rm = TRUE),
        mean_specificity = mean(rs$Spec, na.rm = TRUE)
      )
    )
  }
}

results <- results %>%
  group_by(model) %>%
  mutate(
    delta_AUC = mean_AUC - mean_AUC[feature_set == "full"]
  ) %>%
  ungroup()

The small increases in AUC that we see we have after removing certain features such as telomeric content, TERT, or TF rate are normal and are due to random variation in the data and the training process. This can happen when the removed feature is not very informative for the model, or when its contribution is overshadowed by other features.

Removing TVR motif features causes the largest performance decrease in both models, particularly for Random Forest, suggesting that telomere variant repeat composition contains strong predictive information for ALT status. Removing telomere insertion rate also noticeably reduces performance.

Of course, when removing TVR motifs we are removing 9 individual features instead of one, and thus an abrupt decrease in performance can be expected. Although informative on TVR motifs contribution to classification, the comparison is not fair. What could be interesting is to perform a LOFO on each individual TVR motif, looking for the ones having more discriminatory power.
